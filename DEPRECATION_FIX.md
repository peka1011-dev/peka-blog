# Deprecated 경고 해결 가이드

## 📋 경고 메시지 분석 (초보자용)

### 1. **실제 에러가 아닙니다**
이것들은 **경고(Warning)**입니다. 프로젝트는 실행되지만, 보안이나 호환성 문제가 있을 수 있습니다.

### 2. **경고 종류별 설명**

#### 🔴 **중요한 경고 (수정 필요)**
- `next@14.0.4`: 보안 취약점이 있어 패치된 버전으로 업그레이드 필요
- `eslint@8.57.1`: 더 이상 지원되지 않는 버전

#### 🟡 **무시해도 되는 경고 (하위 의존성)**
- `inflight@1.0.6`: 다른 패키지가 사용하는 하위 의존성 (직접 수정 불가)
- `rimraf@3.0.2`: 다른 패키지가 사용하는 하위 의존성
- `glob@7.1.7`: 다른 패키지가 사용하는 하위 의존성
- `@humanwhocodes/*`: ESLint의 하위 의존성

**왜 수정 안 하나요?**
→ 이들은 우리가 직접 설치한 패키지가 아니라, 다른 패키지들이 내부적으로 사용하는 것입니다.
→ 상위 패키지를 업그레이드하면 자동으로 해결됩니다.

## ✅ 수정 내용

### 변경된 파일: `package.json`

```json
// 변경 전
"next": "14.0.4",
"eslint-config-next": "14.0.4",

// 변경 후
"next": "^14.2.5",           // 보안 패치 적용된 최신 버전
"eslint-config-next": "^14.2.5",  // Next.js 버전과 맞춤
```

### 수정 이유

1. **Next.js 14.0.4 → 14.2.5**
   - 보안 취약점이 패치된 버전
   - Vercel 배포 시 최신 안정 버전 사용 권장
   - `^14.2.5`는 14.2.5 이상 15.0.0 미만의 버전을 자동으로 설치

2. **eslint-config-next 14.0.4 → 14.2.5**
   - Next.js 버전과 일치시켜야 함
   - ESLint 설정이 Next.js 버전에 맞춰져 있음

3. **ESLint는 8.57.1 유지**
   - ESLint 9는 설정 방식이 크게 바뀌어 breaking change가 많음
   - 현재 프로젝트와 호환성을 위해 8.x 유지

## 🚀 적용 방법

```bash
# 1. package.json이 수정되었으므로 의존성 재설치
npm install

# 2. 빌드 테스트 (Vercel 배포 전 확인)
npm run build
```

## 📝 남은 경고들

다음 경고들은 **무시해도 됩니다**:
- `inflight`, `rimraf`, `glob` 등은 하위 의존성이므로 직접 수정 불가
- 상위 패키지들이 업데이트되면 자동으로 해결됨
- 프로젝트 실행에는 영향 없음

## 🔍 Vercel 배포 시 확인사항

Vercel은 자동으로 `npm install`을 실행하므로:
1. ✅ `package.json`의 버전이 업데이트되었는지 확인
2. ✅ 빌드가 성공하는지 로컬에서 테스트 (`npm run build`)
3. ✅ 배포 후 로그에서 경고가 줄어들었는지 확인

## 💡 참고사항

- `^` 기호: 마이너 버전 업데이트를 자동으로 허용 (예: 14.2.5 → 14.3.0)
- `~` 기호: 패치 버전만 업데이트 허용 (예: 14.2.5 → 14.2.6)
- 버전 없이 `^`만 사용: 최신 버전 자동 설치 (권장하지 않음)
